# VERITAS — Updated Data Model Schema

Based on the latest database design (WWW SQL Designer XML export), here are the updated tables and relationships for the VERITAS project. Noticeable changes include the addition of a `countries` table, linking `articles` explicitly to `countries` and `regions` via foreign keys, and switching some `JSONB` fields to `BYTEA` (binary/blob storage).

## 1. users
Supports sign-up, sign-in/out, and admin access control.
| Column | Type | Notes |
| :--- | :--- | :--- |
| `id` | INTEGER | PRIMARY KEY |
| `email` | VARCHAR | |
| `password_digest` | VARCHAR | |
| `role` | VARCHAR | visitor, user, admin |
| `created_at` | TIMESTAMP | |
| `updated_at` | TIMESTAMP | |

## 5. ai_analyses
Stores all LLM-generated outputs for a given article (1-to-1 with articles).
| Column | Type | Notes |
| :--- | :--- | :--- |
| `id` | INTEGER | PRIMARY KEY |
| `article_id` | INTEGER | FOREIGN KEY → articles.id |
| `trust_score` | INTEGER | 0–100 |
| `sentiment_label` | TEXT | e.g. Pro-Trade, Aggressive, Neutral |
| `sentiment_color` | TEXT | e.g. red, green for globe rendering |
| `summary` | TEXT | 2-sentence AI bias/manipulation summary |
| `geopolitical_topic`| TEXT | e.g. Semiconductors, Cyber |
| `threat_level` | TEXT | Normal, Elevated, Severe |
| `linguistic_anomaly_flag`| BOOLEAN | Coordinated propaganda detection |
| `anomaly_notes` | TEXT | Details if flagged |
| `created_at` | TIMESTAMP | |

## 7. narrative_convergences
Stores periodic snapshots of how many outlets are pushing the same storyline.
| Column | Type | Notes |
| :--- | :--- | :--- |
| `id` | INTEGER | PRIMARY KEY |
| `topic_keyword` | TEXT | e.g. Ukraine, Tariffs |

## 8. perspective_filters
Defines available viewpoints for the “Perspective Filter” feature.
| Column | Type | Notes |
| :--- | :--- | :--- |
| `id` | INTEGER | PRIMARY KEY |
| `name` | VARCHAR | e.g. USA, China, EU, Russia |

| `keywords` | TEXT | Terms used to filter feed & globe |

## 9. briefings

| Column | Type | Notes |
| :--- | :--- | :--- |
| `id` | INTEGER | PRIMARY KEY |

| `generated_at` | TIMESTAMP | |
| `threat_summary` | TEXT | |
| `top_narratives` | BYTEA | Snapshot of top stories (Binary) |
| `pdf_url` | VARCHAR | Path/URL to stored PDF file |

## 10. feed_snapshots (Phase 4 — Time Travel Slider)
Periodic snapshots of the globe state to support rewinding up to 48 hours.
| Column | Type | Notes |
| :--- | :--- | :--- |
| `id` | INTEGER | PRIMARY KEY |
| `snapshots_at` | TIMESTAMP | Time this snapshot was taken |
| `articles_payload`| BYTEA | Article IDs + positions active at that time |
| `nodes_payload` | BYTEA | Globe node states at that time |

## Relationship Summary
*   **users** ──< **briefings**
*   **regions** ──< **countries**
*   **regions** ──< **articles**
*   **countries** ──< **articles**
*   **articles** ──< **ai_analyses** (1-to-1)
*   **articles** ──< **narrative_arcs**
