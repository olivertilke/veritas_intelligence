# Narrative Route Generation

Automatically generate narrative routes by finding semantically similar articles using pgvector embeddings.

## How It Works

1. **Article Embeddings**: Each article has a 1536-dimensional vector embedding (generated by `EmbeddingService` during analysis pipeline).

2. **Similarity Matching**: `NarrativeRouteGeneratorService` finds articles with similar embeddings (cosine similarity > 0.75) published within 7 days of each other.

3. **Route Creation**: Similar articles are chained into a `NarrativeRoute` with hops ordered by publication time.

4. **Framing Detection**: Heuristics determine framing shift (original → amplified → distorted) based on source patterns and headline word overlap.

## Usage

### Via Rake Task

```bash
# Generate routes for 50 recent articles
rake veritas:narrative:generate_routes

# Custom limit
rake veritas:narrative:generate_routes LIMIT=100
```

### Via Admin API

```bash
# Trigger via curl (requires admin auth)
curl -X POST http://localhost:3000/admin/narrative_routes/generate \
  -H "Authorization: Bearer YOUR_ADMIN_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{"limit": 50}'
```

### Via Rails Console

```ruby
# Manual trigger
service = NarrativeRouteGeneratorService.new
service.generate_routes(limit: 50)

# Check statistics
NarrativeRoute.count
NarrativeRoute.where(is_complete: true).count
```

## Background Job

The job is queued automatically after article fetching, or can be run manually:

```ruby
GenerateNarrativeRoutesJob.perform_later(limit: 50)
```

## Algorithm Details

### Similarity Threshold
- **Cosine distance**: Articles with distance < 0.25 (similarity > 0.75) are considered related.
- **Time window**: Articles must be published within 7 days of the origin article.

### Hop Construction
- Hops are ordered by `published_at`.
- `delay_from_previous` calculated as seconds between consecutive articles.
- Maximum 8 hops per route (configurable via `MAX_HOPS_PER_ROUTE`).

### Framing Shift Detection
- **original**: First article in chain
- **amplified**: State media (RT, Sputnik, Xinhua) or partisan outlets (Breitbart, CNN, MSNBC)
- **distorted**: Headline word overlap < 40% (Jaccard similarity)
- **neutralized**: Default for other cases

### Confidence Score
Calculated from embedding similarity, normalized to 0.5–0.95 range.

## Troubleshooting

### No Routes Generated
- Ensure articles have embeddings: `Article.where.not(embedding: nil).count`
- Check similarity threshold: lower `SIMILARITY_THRESHOLD` constant if too strict

### Job Fails with Timeout
- Reduce `limit` parameter
- Increase Solid Queue timeout for `default` queue

### Routes Have Incorrect Framing
- Review `detect_framing_shift` heuristics in service
- Consider adding custom source pattern matching
